An Introduction to Git
Why?.

Chances are you have heard or worked with Git. It is one of the most popular code management systems in the world. But it can be quite daunting to comprehend the inner workings of this system, especially if you are collaborating with others and working on multiple branches.

This project's goal is to essentially demystify the Git version control system. If you are new to Git and if you ever wondered about the underlying process, you're at the right place.

Scroll down to learn about the Git Content Model, an intuitive way to understand basic Git commands and play with the interactive visualization.

So, what is Git?
Git is a type of version control system (VCS) that makes it easier to track changes to files. For example, when you edit a file, git can help you determine exactly what changed, who changed it, and why.

What makes git unique?
  • Git branches are cheap and easy to merge. This facilitates the feature branch workflow popular with many Git users.
  • Git is a distributed version control system. Every user gets their own local repository, complete with a full history of commits.
  • Having a full local history makes Git fast, since it means you don’t need a network connection to create commits, inspect previous versions of a file, or perform diffs between commits.
Git Location Model
Git Structure

Where does git store your code. Contrary to what you'd think, it's not just your computer and the cloud. In Git, there are five places your source can exist:
  • A Stash
  • A Local working directory.
  • An Index (or staging area)
  • a local repository
  • A remote repository
