An Introduction to Git
Why?.

Chances are you have heard or worked with Git. It is one of the most popular code management systems in the world. But it can be quite daunting to comprehend the inner workings of this system, especially if you are collaborating with others and working on multiple branches.

This project's goal is to essentially demystify the Git version control system. If you are new to Git and if you ever wondered about the underlying process, you're at the right place.

So, what is Git?
Git is a type of version control system (VCS) that makes it easier to track changes to files. For example, when you edit a file, git can help you determine exactly what changed, who changed it, and why.

What makes git unique?
  • Git branches are cheap and easy to merge. This facilitates the feature branch workflow popular with many Git users.
  • Git is a distributed version control system. Every user gets their own local repository, complete with a full history of commits.
  • Having a full local history makes Git fast, since it means you don’t need a network connection to create commits, inspect previous versions of a file, or perform diffs between commits.
Git Location Model
Git Structure

Where does git store your code. Contrary to what you'd think, it's not just your computer and the cloud. In Git, there are five places your source can exist:
  • A Stash
  • A Local working directory.
  • An Index (or staging area)
  • a local repository
  • A remote repository
Workspace
This is the local working directory for your local repository whose contents are checked out in the branch you are working in.
Index
These include the files you want to commit. This is considered the staging area where the files you Before you “commit” (checkin) files, you need to first add them to the index. Also called "current directory cache", "staging area", "cache" or "staged files".
Local Repository
Git local repository is the one on which we will make local changes, typically this local repository is on our computer. It contains a subdirectory named `.git` that contains all of your necessary repository files — a Git repository skeleton. The default name for a local branch is "master".
Remote Repository
Git remote repository is a remotely hosted service which stores a version(s) of your code, ensuring all your changes are available for other developers. The default name for a remote is "origin".
Stash
This is a place where you can store your code changes while you are trying to pull in the latest version from the remote repository.
Learn about some Git Commands
As we have seen above our source code are usually physically hosted on two machines: a local and a remote repo. While collaborating with people on the same repository, we regularly encounter situations when we need to either update our local repository to take up changes that someone else has made. Similarly, we also push our code to the remote repository when we want to update the codebase so others can access the new version.
